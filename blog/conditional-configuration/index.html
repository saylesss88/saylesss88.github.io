Therefore, if <code>config.custom.hyprland.enable</code> evaluates to true, the wlogout module will be enabled by default.</li> </ul> <h2 id="benefits-of-this-approach">Benefits of this Approach</h2> <p>The <code>lib.mkIf cfg.enable</code> ensures that wlogoutâ€™s configuration (e.g., enabling <code>programs.wlogout</code>) is only applied when <code>custom.wlogout.enable = true</code>, which defaults to <code>custom.hyprland.enable</code>. This means wlogout is enabled by default only if Hyprland is enabled, but I can override this (e.g., <code>custom.wlogout.enable = true</code> without Hyprland). This conditional logic prevents wlogout from being installed unnecessarily when Hyprland is disabled, unlike a simpler approach like <code>programs.wlogout. enable = config.programs.hyprland.enable</code>, which hardcodes the dependency and offers less flexibility.</p>
